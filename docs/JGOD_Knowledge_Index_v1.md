### 第一階段：核心法則（必讀）

   - **重點**: Walk-Forward Analysis、Governance 規則評估

   - **為什麼讀**: Governance 是 J-GOD 的核心價值，所有策略必須通過 Governance 檢查才能被採用
   - **重點**: Sharpe、MaxDD、TE、Turnover 等規則的判斷標準

4. **docs/JGOD_System_Map_v1.md**
   - **為什麼讀**: 提供系統的鳥瞰視圖，幫助理解各模組的職責與關係
   - **重點**: 模組地圖、資料流、檔案對應表

   - **重點**: 閉環系統設計、自主演化邏輯

## 🔍 文件分類說明

### ACTIVE
- **定義**: 當前活躍使用的文件，內容準確且與實作一致
- **使用建議**: 優先閱讀，可作為開發與決策的參考

### LEGACY
- **定義**: 舊版本文件，已被新版本取代，但保留作為歷史參考
- **使用建議**: 僅在需要了解歷史演進時閱讀

### DRAFT
- **定義**: 規劃或草稿文件，內容可能不完整或未實作
- **使用建議**: 僅在了解未來規劃時閱讀

## 📝 文件位置說明

- **docs/**: 主要文件目錄（標準文件、使用說明）

## 🔄 文件更新政策

1. **ACTIVE 文件**: 應與實作保持同步，任何實作變更應同步更新文件
2. **LEGACY 文件**: 保留不刪除，但標記為 LEGACY
3. **DRAFT 文件**: 當規劃實作完成後，應轉為 ACTIVE 或刪除
